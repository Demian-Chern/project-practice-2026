# Отчет по индивидуальному заданию: Разработка децентрализованного блокчейна на языке Python

**Цель работы:** Проектирование и программная реализация учебной модели распределенного криптографического реестра (блокчейн) с механизмами консенсуса Proof of Work и транзакционной безопасностью на основе асимметричного шифрования RSA.

## 1. Архитектура и технические компоненты системы
Реализованная система разделена на два изолированных программных слоя:
1. **Blockchain Client (Клиентский узел):** Отвечает за генерацию криптографических адресов (кошельков) при помощи алгоритма RSA с длиной ключа 1024 бит, а также за формирование структуры транзакции и её подписание приватным ключом с помощью схемы PKCS1_v1_5 и хеширования SHA-1.
2. **Blockchain Miner/Node (Узел сети/Майнер):** Выступает в роли распределенной базы данных, группирующей входящие транзакции в блоки.

## 2. Ключевые алгоритмы и функции реестра
* **Связывание блоков (Chaining):** Каждый последующий блок в структуре `chain` агрегирует внутри себя криптографический SHA-256 хеш метаданных предыдущего блока (`previous_hash`). Любое несанкционированное изменение исторических данных в промежуточных блоках мгновенно нарушает валидность всех последующих дочерних звеньев сети.
* **Proof of Work (Доказательство выполнения работы):** Алгоритм подбора числового значения `nonce`, при котором итоговый хеш блока удовлетворяет текущему критерию сложности (в нашей реализации — наличие `MINING_DIFFICULTY = 4` ведущих нулей). Процесс требует вычислительных мощностей, что защищает сеть от спама и атаки Сивиллы.
* **Сетевой консенсус (Conflict Resolution):** Сеть функционирует без единого доверенного центра. При возникновении конфликтов синхронизации (когда два майнера одновременно добывают блок) применяется правило "Longest Chain Wins" — локальная копия реестра заменяется цепочкой контрагента только в том случае, если она длиннее текущей и успешно прошла поблочную валидацию.
